# ADR-001: Immutable User Configuration

**Status:** Accepted

## Context

NixOS supports both mutable (imperative) and immutable (declarative) user management. Mutable users allow `useradd` and `passwd` on the host but create drift between what Nix declares and what is on disk.

## Decision

`users.mutableUsers = false`. All user accounts and authorized SSH keys for interactive login are declared in `modules/users.nix`. No password-based login; authentication is SSH key only. Exception: the initrd SSH server (used for remote ZFS unlock) has its own `authorizedKeys` entry in `hosts/hermes/hardware.nix` under `boot.initrd.network.ssh` — that key is not in `modules/users.nix` and must be rotated there separately.

## Consequences

No users can be added imperatively on the host. Adding a user requires a NixOS rebuild. This eliminates an entire class of configuration drift. Key rotation requires updating both `modules/users.nix` (interactive SSH) and `hosts/hermes/hardware.nix` (initrd SSH) if both access paths need to be changed.
