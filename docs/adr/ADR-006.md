# ADR-006: Schedutil CPU Governor

**Status:** Accepted

## Context

The Linux kernel offers several CPU frequency scaling governors. `performance` locks frequency at maximum, `powersave` locks at minimum, and `schedutil` adjusts frequency based on scheduler load signals.

## Decision

Use the `schedutil` governor. It responds to load changes faster than `ondemand` and is more efficient than `performance` while remaining more responsive than `powersave`.

## Consequences

CPU frequency adapts dynamically. This gives better efficiency during idle periods and near-peak performance under load. Frequency transitions introduce minor latency that would not exist with `performance`, which is acceptable for this workload.
