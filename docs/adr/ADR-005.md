# ADR-005: NVMe Power Management Disabled

**Status:** Accepted

## Context

Modern NVMe drives support multiple power states with varying latency profiles. The kernel may park drives in low-power states to save energy, introducing latency spikes on the first I/O after idle periods.

## Decision

`nvme_core.default_ps_max_latency_us=0` disables NVMe power state transitions, keeping drives at full performance at all times.

## Consequences

Idle power draw is slightly higher. I/O latency remains consistent and predictable, which is important for agent workloads that interleave idle periods with bursts of disk activity.
