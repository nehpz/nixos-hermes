# ADR-003: Dual ESP With Rsync Replication

**Status:** Accepted

## Context

ZFS is mirrored across two NVMe drives, protecting data. However, the EFI System Partition (ESP) is not part of the ZFS pool and is typically on a single drive, creating a single point of failure for booting.

## Decision

Each NVMe carries a 1GB FAT32 ESP. The primary mounts at `/boot`; the secondary mounts at `/boot-fallback`. On every `nixos-rebuild switch`, systemd-boot replicates the primary ESP to the fallback via `rsync`.

## Consequences

A single drive failure does not prevent booting. The replication is triggered by NixOS activation, not automatically on every write, so a failure between activations leaves the fallback one generation behind. Caveat: `/boot-fallback` is mounted with `nofail`; if the secondary ESP fails to mount and the rsync still runs, it silently copies into the root filesystem directory rather than onto the disk, giving false confirmation of success. The redundancy guarantee only holds when both drives are healthy at activation time.
